## Day 2 核心要点

### 具体写

#### 标点符号

两个独立意思，用**句号**，不用逗号

两个句子相关联，用**分号**

建议，不用破折号

#### 词

##### 定义新术语

+ 开头定义时，**缩短**

+ 后文使用中，**一致**

##### 首字母缩略

缩略条件：1.明显短 2.多次用

+ 第一次出现时，用**括号**

​	*例如，Telekinetic Tactile Network (TTN)* 

+ 后文使用中，**一致**（不来回切换）

##### 代词

+ 在名词**后**出现

+ **靠近**名词（不超过5个单词）

+ 有**第二个名词**时，**不使用**代词

#### 句

##### 段落开头句

一句话，**写清段落主旨**

##### 清晰句子

+ **主动语态**

+ **强动词**==（需要自己多看，整理词汇库==

>是
>
>引发>出现
>
>生成>发生
>
>确保 
>
>……

+ **祈使动词**，暗示主语“你”

+ **祈使名词**，不用“有一个”

+ 少用形容词和副词（因为不精确

##### 短句

+ 缩短语（可用一个词，替代一个短语

+ 切短句

+ 多逗号

+ 用列表

#### 表

##### 列表

+ 列表内容，要**同类/同句子结构**

+ 如果是**句子**，用**句号**

##### 表格

+ **一句话**加冒号，介绍表格
+ 好理解的，有用的**标题**

+ 内容**同类型**

+ **少文字**，少于两个句子（否则用其他形式

### 大方向

#### 受众

+ **定义受众**

​	角色/知识水平内容/受众目标

​	受众之前已经知道什么

​	受众之后能做什么

+ **明确内容**

​	罗列任务

​	*例如 受众能知道XX原因*

​		 *受众能操作XX*

#### 文档

+ 说明文档范围

​	*例如 本文档介绍了XXX的设计*

 		*本文档**不表述**XXX的设计*（不符合受众期待，才需特别强调）

+ 说明受众

  *例如 本文档面向以下受众：*

  ​	 *本文档假设您了解XXX*

  ​	 *在阅读本文档之前，您必须阅读XXX*

+ 大纲目录

+ 一个标题：简短/写清你想表达的要点

+ 拆分，复杂图

#### 更好的使用大模型语言(LLM)

+ 怎么写

1.定义角色

2.定义受众

3.文档类型（详细，不能太泛）

4.目标

5.一些其他附加条件

例如

1.你是一位专业的技术作家

2.向XXX介绍XX应用程序

3.生成XX中的XX部分

4.阅读完后，读者能知道XX

5.语气严肃，要简短……

+ 写摘要

例如 写一篇关于所附课程的一句话总结。将摘要针对软件工程师，他们将使用摘要来决定是否参加该课程。使摘要引人入胜，但又专业;摘要不应看起来像营销文献。
